# Procore Content Design Skill — How to Use

This skill encodes **Procore's content design standards** so designs and copy created with AI tools stay consistent with voice, terminology, grammar, accessibility, and global/inclusive guidelines.

---

## What's included

- **SKILL.md** (skill root) — Main instructions, workflows, and review checklist
- **references/reference.md** — Detailed tables (glossary, field labels, dates, measurements, translation)
- **references/master-glossary-contexts.md** — Context-bound terminology map for Master Glossary usage
- **references/README.md** — This file (human guide)

---

## Using this in Cursor

1. Keep this folder at `.cursor/skills/procore-content-design/` in your repo.
2. Ask for content work such as drafting, review, or conversation design.
3. Use direct prompts like "Apply Procore content design standards to this copy."

---

## Testing prompts

1. Review copy:
   - "Review this UI copy against Procore content design standards and fix issues."
2. Generate copy:
   - "Write an empty state and confirmation modal for inspection reports."
3. Conversation flow:
   - "Design a short AI assistant flow to help users find Timesheet."

---

## Maintenance

Update `SKILL.md` for workflow/rules and `references/reference.md` for detailed lookups. For official terminology changes, coordinate with `#ux-content`.
